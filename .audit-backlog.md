## MEDIUM (pending — реальные баги, требуют внимания)

- [ ] **M1 — `dispatchQueuedItems` FIFO under concurrent re-enqueues.**
  File: `core/src/main/kotlin/ru/sipaha/spkremote/core/RemoteClient.kt:766-826`.
  Status: **partially fixed**. Backlog drain agent попытался sequential drain (await каждого `call()` в lifecycle coroutine), но это блокировало lifecycle channel → deadlock в тестах + OOM. Откатил на per-item `scope.launch`. Per-item launch сохраняет FIFO при последовательных отказах, но НЕ при concurrent (если транспорт упал во время первого call, и второе уже залогировано в launch — второе может re-enqueue раньше первого).
  Test T1 (`mid-flush drop re-enqueues remaining items at head in FIFO order`) — `@Disabled` со ссылкой на этот пункт.
  Правильный фикс: option (b) из Phase 3 review — accumulate failed items in a CompletableDeferred list, then re-prepend in original index order via single synchronized block AFTER all launches complete. Требует переписать concurrency model в `dispatchQueuedItems`.
